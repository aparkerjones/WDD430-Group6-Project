# W02 Group Project Scope Summary

## Purpose
Build a full-stack web application as a team while practicing:
- Collaborative development workflow in GitHub
- Project planning using a board and work items
- UI theme and component design before implementation
- Professional team communication and leadership

## Requirements from Week 02 Group Project 02
1. Meet as a group and discuss requirements/scope.
2. Create one public GitHub repository for the group project.
3. Create a GitHub Project linked to that repository (kanban style).
4. Invite group members as collaborators to repository and project.
5. Add a project README including group member names.
6. Each member clones the repository locally.
7. Produce initial design direction:
- color scheme/theme
- typography
- layout ideas
- initial component thinking
8. Brainstorm user stories/work items and add at least 10 items to project board.
9. Select next group leader.

## Limitations and Constraints
- Final product is not expected this week; focus is setup/planning/design.
- Work items are expected to evolve over upcoming weeks.
